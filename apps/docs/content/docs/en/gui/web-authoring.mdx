# Web Authoring

A3S GUI starts from a Web developer experience: application authors write
React/TSX and React Aria-shaped components, then the bridge lowers that semantic
tree into native controls.

The intended TypeScript surface is ordinary TSX:

```tsx
/** @jsxImportSource @a3s-lab/gui */
import {Button, Input, Label, TextField} from 'react-aria-components';
import {createAction, createUiFrame, defineAction} from '@a3s-lab/gui';

const setEmail = createAction('setEmail', 'Set email');
const saveProfile = createAction('saveProfile', 'Save profile');

const root = (
  <form className="profile-form" data-screen="profile">
    <TextField name="email" isRequired>
      <Label>Email</Label>
      <Input
        placeholder="you@example.com"
        style={{minWidth: 280}}
        onChange={setEmail}
      />
    </TextField>

    <Button aria-label="Save profile" onPress={saveProfile}>
      Save
    </Button>
  </form>
);

export const frame = createUiFrame('profile', root, {
  window: {title: 'Profile', width: 640, height: 480},
  actions: [defineAction(setEmail), defineAction(saveProfile)],
});
```

The application author should not import native widget names or write Rust UI
code for normal screens.

## Compiler Bridge Shape

The React Compiler integration lowers TSX into a serializable tree carrying the
same information as JSX props:

- React Aria component identity
- stable keys for reconciliation
- text children
- `className`
- inline `style`
- `aria-*`, `data-*`, and portable HTML attributes
- event props such as `onClick`, `onChange`, and `onPress` with named callbacks
- React Aria state props such as `isDisabled`, `isSelected`, `isRequired`, and
  `isInvalid`
- HTML and ARIA aliases such as `disabled`, `required`, `aria-expanded`,
  `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`

The Rust core maps that tree through `ReactCompilerBridge`, then renders it with
`GuiRuntime` and the keyed `Renderer`.

## Native Lowering Rules

| Web-authored input | Native IR output |
| --- | --- |
| `className` | Preserved in `WebProps.class_name` for style resolution. |
| `style={{...}}` | Preserved in `WebProps.style` and parsed into `PortableStyle`. |
| `aria-label` | Used as the native accessibility label when no explicit or visible label exists. |
| `aria-*` | Preserved as accessibility metadata; supported state attributes also feed native control state. |
| `data-*` | Preserved as metadata for tests, analytics, and automation. |
| `disabled` / `required` / `checked` / `selected` | Normalized to React Aria-style native control state. |
| `min` / `max` / `aria-valuenow` | Normalized to native ranged control state. |
| `onClick` / `onPress` | Normalized to the primary native action. |
| `onChange` | Normalized to the primary action for value controls. |
| `TextField` + `Label` + `Input` | Folded into one native text field. |
| `Select` + `ListBoxItem` | Folded into a native select with native options. |

React Aria's `onPress` is preferred over `onClick` when both are present, so
authors can use idiomatic React Aria APIs without losing native press semantics.

## Portable Contract

The syntax is Web-compatible. The runtime is native. Code that assumes a live
browser DOM, CSSOM, browser layout APIs, or concrete `HTMLElement` instances is
outside the portable contract and should be caught before runtime.
