# Multi-Machine

A3S Code supports multi-machine architectures through explicit external or hybrid dispatch. The runtime does not require a queue for local sessions, and local subagent delegation does not require lane queues.

## Pattern

```text
coordinator session
  -> task or lane dispatch
  -> external worker
  -> result, changed files, verification evidence
  -> coordinator merge
```

## Coordinator

The coordinator owns the conversation, final synthesis, application policy, and
release decision. It can expose selected queue lanes to external workers.

## Worker

Workers should receive bounded task payloads and return compact results:

- summary
- files inspected or changed
- commands run
- verification status
- errors and retry state

## Transport

Choose the transport that fits your platform: HTTP, message queues, job systems, or internal RPC. A3S Code's lane queue APIs are the integration point; the deployment topology is yours.
