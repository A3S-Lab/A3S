# A3S GUI

`a3s-gui` 是将结构化 UI 输入转换为 native UI IR 和 platform command stream
的 Rust crate。它接受 `UiFrame` protocol frames、JSX-generated element records
和 `NativeElement` trees；platform adapters 将 command stream 应用到 AppKit、
WinUI 或 GTK。

crate metadata 指向 [`A3S-Lab/GUI`](https://github.com/A3S-Lab/GUI)，并作为
`crates/gui` submodule 固定在当前 monorepo 中。

## 能力地图

| 领域 | 当前能力 |
| --- | --- |
| Input model | 支持 JSX-generated element records、React Aria semantic identifiers、HTML/SVG intrinsic names、DOM-style props、inline style、aria/data attributes 和命名事件回调。 |
| Native IR | Typed tree，包含 native role、state、label、value、accessibility metadata、style token、event 和 children。 |
| Runtime | `GuiRuntime`、keyed renderer diff engine、action registration、event routing，以及用于验证的 headless recording backend。 |
| Command stream | 可序列化 native command，例如 create、update、insert child、remove、set root。 |
| Platform planning | AppKit、WinUI、GTK planning adapter 将 IR 映射为 native widget blueprint。 |
| Native paths | crate manifest 中存在 feature-gated AppKit、WinUI、GTK native adapter paths。 |
| TypeScript bridge | `sdk/typescript` 下有原型 TypeScript SDK，提供 JSX/runtime protocol helper。 |

## 边界

Native feature paths 从 command stream 创建平台控件，不嵌入 browser engine。A3S WebView
是单独的组件，用于 embedded web content。
