# TCP Filter

`tcp-filter` 是连接级过滤器，用于 TCP 入口点。它不属于 HTTP 管道中间件，也不能写成 `middlewares "<name>" { type = "tcp-filter" }`。它通过入口点字段启用：`max_connections` 和 `tcp_allowed_ips`。

## 适用场景

- 代理数据库、消息队列、私有 TCP 服务时限制并发连接。
- 只允许内网、办公网或特定服务网段连接 TCP 入口。
- 在应用协议之前先做成本很低的连接准入控制。

## 工作方式

连接到达 TCP 入口点时，过滤器先检查 IP 白名单，再检查并发连接数：

1. 如果 `tcp_allowed_ips` 非空，客户端 IP 必须匹配其中一个 IP/CIDR。
2. 如果配置了 `max_connections`，当前在途连接数必须小于上限。
3. 连接被接受后会持有一个 RAII permit，连接结束时自动递减计数。

被 IP 拒绝的连接不会占用连接计数。超过连接数上限会返回内部错误信息 `TCP connection limit reached`。

## 配置

```text
entrypoints "tcp-db" {
  address         = "0.0.0.0:5432"
  protocol        = "tcp"
  max_connections = 1000
  tcp_allowed_ips = ["10.0.0.0/8", "192.168.1.0/24"]
}
```

| 键 | 类型 | 默认值 | 说明 |
|-----|------|---------|-------|
| `max_connections` | u32 | _未设置表示不限_ | 最大并发 TCP 连接数。 |
| `tcp_allowed_ips` | array&lt;string&gt; | `[]` | 空数组表示允许所有 IP；支持单个 IP 和 CIDR。 |

## 与 HTTP 中间件的区别

| 项 | HTTP 中间件 | TCP Filter |
|----|-------------|------------|
| 配置位置 | `middlewares "<name>"` | `entrypoints "<name>"` |
| 引用方式 | `routers.middlewares` | 入口点字段直接生效 |
| 处理对象 | HTTP 请求/响应 | TCP 连接 |
| 典型返回 | `401` / `403` / `429` / `503` | 连接被拒绝 |

## 示例

```text
entrypoints "redis-tcp" {
  address         = "0.0.0.0:6379"
  protocol        = "tcp"
  max_connections = 200
  tcp_allowed_ips = ["10.42.0.0/16"]
}
```

## 注意事项

- `tcp_allowed_ips = []` 表示允许所有 IP；这与 `ip-allow` 的空列表报错不同。
- `max_connections` 不设置表示不限流；设置为较小值可保护后端，但也可能拒绝正常峰值。
- TCP 入口没有 HTTP 状态码，客户端只会感知连接失败或被关闭。
- 如果前面还有四层负载均衡，需要确认网关看到的是最终客户端 IP 还是负载均衡器 IP。

## 推荐做法

生产 TCP 入口建议至少配置一项保护：

```text
max_connections = 1000
tcp_allowed_ips = ["10.0.0.0/8"]
```
