# CORS

`cors` 中间件处理浏览器跨域访问。它会直接响应 `OPTIONS` 预检请求，并在普通响应上添加 CORS 相关响应头。

## 适用场景

- Web 前端从不同域名访问 A3S Gateway 后面的 API。
- 需要集中管理允许来源、方法、请求头和预检缓存时间。
- 希望后端业务服务不再重复实现 CORS。

## 工作方式

对于 `OPTIONS` 请求，中间件读取 `Origin`：

- 来源允许：返回 `204`，并带上 `Access-Control-Allow-*` 头。
- 来源不允许：返回 `403`，响应体为 `Origin not allowed`。

对于非预检请求，中间件在响应方向添加 `Access-Control-Allow-Origin`。如果 `allowed_origins` 包含 `*`，则输出 `*`；否则输出配置列表的第一项。

## 配置

```text
middlewares "cors" {
  type            = "cors"
  allowed_origins = ["https://console.example.com", "https://app.example.com"]
  allowed_methods = ["GET", "POST", "PUT", "DELETE"]
  allowed_headers = ["Content-Type", "Authorization"]
  max_age         = 3600
}
```

| 键 | 类型 | 默认值 | 说明 |
|-----|------|---------|-------|
| `type` | string | _必填_ | 固定为 `cors`。 |
| `allowed_origins` | array&lt;string&gt; | `["*"]` | 支持 `*` 或精确来源匹配。 |
| `allowed_methods` | array&lt;string&gt; | `["GET","POST","PUT","DELETE","OPTIONS"]` | 输出为 `Access-Control-Allow-Methods`。 |
| `allowed_headers` | array&lt;string&gt; | `["Content-Type","Authorization"]` | 输出为 `Access-Control-Allow-Headers`。 |
| `max_age` | u64 | `86400` | 预检缓存秒数。 |

## 示例

```text
middlewares "console-cors" {
  type            = "cors"
  allowed_origins = ["https://console.example.com"]
  allowed_methods = ["GET", "POST", "PATCH", "DELETE"]
  allowed_headers = ["Content-Type", "Authorization", "X-Request-Id"]
  max_age         = 600
}

routers "api" {
  rule        = "Host(`api.example.com`)"
  service     = "api"
  middlewares = ["console-cors", "auth-jwt"]
}
```

## 注意事项

- CORS 是浏览器安全策略，不是服务端认证；仍然需要 `jwt`、`forward-auth` 或其他认证中间件。
- 如果不想开放所有来源，不要使用默认 `["*"]`，显式列出前端域名。
- 非预检响应不会根据请求 `Origin` 动态选择来源；非通配配置会返回第一项来源。
- 预检请求会被 CORS 中间件短路，因此通常把 `cors` 放在认证前，让浏览器能完成预检。

## 推荐顺序

```text
middlewares = ["cors", "auth-jwt", "rate-limit", "headers"]
```
