# Rate Limit

`rate-limit` 是进程内令牌桶限流器。它不依赖外部存储，配置简单、延迟低，适合单实例网关或每个实例都可以独立限流的场景。

## 适用场景

- 给内部 API、管理 API 或低风险公开 API 设置基础吞吐保护。
- 限制突发请求，避免后端被瞬时流量打满。
- 在开发、测试、单节点部署中快速启用限流。

## 工作方式

令牌以 `rate` 个/秒的速度补充，桶容量为 `burst`。每个请求消耗 1 个令牌；令牌不足时短路请求。

失败时返回 `429`，并携带：

```text
Retry-After: 1
```

响应体：

```json
{"error":"Rate limit exceeded"}
```

当前实现是中间件实例的**单个共享桶**，不是按 IP、用户或租户隔离的桶。

## 配置

```text
middlewares "rate-limit" {
  type  = "rate-limit"
  rate  = 100
  burst = 50
}
```

| 键 | 类型 | 默认值 | 说明 |
|-----|------|---------|-------|
| `type` | string | _必填_ | 固定为 `rate-limit`。 |
| `rate` | u64 | _必填_ | 每秒补充的令牌数；缺失会导致配置加载失败。 |
| `burst` | u64 | `rate` | 桶容量和初始令牌数。 |

## 示例

```text
middlewares "public-api-limit" {
  type  = "rate-limit"
  rate  = 50
  burst = 100
}

routers "public-api" {
  rule        = "Host(`api.example.com`)"
  service     = "api"
  middlewares = ["public-api-limit"]
}
```

## 调参建议

- `rate` 控制长期平均吞吐。
- `burst` 控制可接受的瞬时峰值；若希望短时间突发更平滑，可让 `burst` 大于 `rate`。
- 如果网关有多个副本，总体可承载峰值约等于每副本桶容量之和；需要全局限流时使用 [Redis Rate Limit](/cn/docs/gateway/middleware/rate-limit-redis)。

## 推荐顺序

通常放在认证之后，避免匿名恶意流量消耗共享桶：

```text
middlewares = ["ip-allow", "auth-jwt", "rate-limit", "body-limit"]
```
