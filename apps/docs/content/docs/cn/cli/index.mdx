# A3S CLI

`a3s` 是 A3S 栈的统一命令。它把交互式编程智能体终端体验打包成一个入口，
并把产品命令转发给对应组件二进制。

当前 crate version 是 `a3s` `0.9.1`。它的 repository metadata 指向
[`A3S-Lab/Cli`](https://github.com/A3S-Lab/Cli)。

## 它负责什么

| 表面 | 能力 |
| --- | --- |
| `a3s code` | 启动交互式编程智能体 TUI，底层由 `a3s-code-core` 驱动，由 `a3s-tui` 渲染。 |
| `a3s code resume <id>` | 从当前 workspace 恢复已保存的 TUI session。 |
| `a3s box ...` | 转发到 `a3s-box ...`，并可在首次使用时引导安装 Box 运行时。 |
| `a3s use ...` | 第一次真正使用时安装已校验的 A3S Use release，随后提供 Browser、Document、原生 Office、本地 OCR、Box 与已安装 extension route。 |
| `a3s list` | 只读列出 typed component catalog、本地 readiness 与已注册外部工具。 |
| `a3s install/upgrade/uninstall <component>` | 仅管理已注册且有 ownership receipt 的 A3S component 与 delegated Use runtime。 |
| `a3s top` | 打开 live process/container/agent monitor。 |
| `a3s update` | 检查最新 CLI release，并通过 Homebrew 或 standalone release asset 升级。 |
| `a3s <tool> --help` | 把 help 请求转发给对应工具。 |
| TUI 内 `/update` | 从 Homebrew 或 release assets 更新 CLI，并验证安装后的目标版本。 |

## 安装

```bash
brew install A3S-Lab/tap/a3s
cargo install a3s
cargo install --git https://github.com/A3S-Lab/Cli
```

## `a3s code`

终端智能体不是托管服务。它是一个嵌入 A3S Code 的应用：消费
`AgentEvent` 流，并在终端里提供模型、配置、Git、记忆、文件、插件、更新和
A3S OS 视图等控制。

它可以使用普通 ACL 模型配置；在宿主明确接入时，也可以展示账号模型标签页。
CLI 会让宿主工具继续走 A3S Code 适配路径，而不是把任意工具执行交给另一个运行时。

## 首次使用的应用能力

安装 `a3s` 时不会立刻下载 A3S Use。第一次执行 `a3s use ...`，或启动
`a3s code` TUI 时，只要网络和 first-use policy 允许，CLI 就会安装完整且已校验的
Use release。`--offline`、`A3S_OFFLINE=1` 与 `A3S_NO_AUTO_INSTALL=1`
都是严格的禁止下载边界。

该 release 已包含 Browser driver 与 Skills、原生 Office、Document/OCR Skills，
以及 PP-OCRv6 Small ONNX 模型。之后按真正需要准备 runtime：

- Browser 复用系统 Chrome，或准备共享的
  `~/.local/share/a3s/use/browser` cache。A3S Search 使用同一个
  `a3s-use-browser` cache，不会再装一份 Chrome。
- 原生 DOCX、XLSX、PPTX Office 操作不依赖外部应用；只有尚未 native 化的
  compatibility command 才会准备可选 OfficeCLI。
- OCR 与 Document 共用 release 中的 PP-OCRv6 模型；如果 source/精简安装缺少模型，
  第一次 OCR 提取或真正选择 raster evidence 的 Document parse 才会按 policy 准备它。
- PP-OCRv6 通过 ONNX Runtime 完全本地运行，不需要 Tesseract、Python、
  PaddlePaddle、Microsoft Office、LibreOffice、Browser 或远程 OCR 服务。

```bash
a3s use browser open https://example.com
a3s use office native create report.docx --json
a3s use document inspect report.docx --json
a3s use document parse report.docx --ocr auto --json
a3s use ocr extract scan.png --json
```

## 边界

- CLI 是打包和 UX 胶水；agent loop 属于 A3S Code。
- `a3s-box`、`a3s-webview` 等是独立 helper 二进制。
- 本地凭据和账号状态是本地运行态，不应进入仓库文档或示例。
